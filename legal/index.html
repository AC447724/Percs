Privacy Policy

Data Handling

01. Information Collection

To provide high-quality moderation and utility, we store essential metadata including:

  • Server IDs and channel configurations for command routing.
  • User IDs for level tracking and economy balance persistence.
  • Temporary behavioral logs to identify and mitigate raid attempts.

02. Stateless Processing

We utilize a stateless processing architecture for all incoming events. Message data utilized by our security heuristics and content filters is treated as a transient stream; this data is never committed to persistent storage disks and is purged immediately following the completion of the requested moderation logic.

03. Data Security

All sensitive configuration data is encrypted using AES-256-GCM. Access to our database is restricted to authorized systems, protected by physical hardware security modules (HSM) and private entropy keys.

04. Data Deletion

Owners may request a full purge of their server data by removing the bot and contacting our support hub. Data is automatically flagged for deletion 30 days after the bot leaves a guild.
